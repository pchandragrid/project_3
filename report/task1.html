Task 1 – BLIP Optimization (Memory-Efficient Training + CoreML Export)

This task fine-tunes BLIP on a subset of COCO captions, exports to ONNX, converts to CoreML, quantizes, and benchmarks latency/quality.

What this task does

1) Memory-efficient fine-tuning

Uses gradient checkpointing and mixed precision for forward passes (fp16) with careful loss handling. Training resolution is 224px.

2) Exports to ONNX

Exports encoder + decoder with dynamic axes so exported runtime supports variable token lengths.

3) Converts to CoreML + quantizes

Converts to CoreML using a recommended configuration (CPU_AND_NE) and applies 4-bit quantization.

4) Benchmarks

Compares PyTorch (full precision) vs quantized CoreML on latency and BLEU-4.

How to run (step-by-step)

1) Train

python -m tasks.task1_blip_optimization.train_blip_memory_efficient \
  --annotation_path src/data/processed/subset_10k.jsonl \
  --image_dir src/data/raw/train2017 \
  --batch_size 4 \
  --image_size 224

2) Export ONNX

python -m tasks.task1_blip_optimization.export_onnx

3) Convert to CoreML + quantize

python -m tasks.task1_blip_optimization.convert_coreml \
  --compute_units CPU_AND_NE \
  --conversion_mode torch

4) Benchmark

python -m tasks.task1_blip_optimization.benchmark \
  --compute_units CPU_AND_NE \
  --num_samples 100

Key outputs

  • Checkpoint: tasks/task1_blip_optimization/checkpoints/blip_gc_mp_224/
  • ONNX exports: tasks/task1_blip_optimization/exports/onnx/
  • CoreML exports: tasks/task1_blip_optimization/exports/coreml/
  • Benchmark files: tasks/task1_blip_optimization/results/coreml_benchmark.json and tasks/task1_blip_optimization/results/coreml_benchmark.md

Observed results (current repo run)

Evaluated on 100 samples

Model Mean Latency (ms) Std (ms) BLEU-4
PyTorch (full precision) 258.67 347.66 0.1814
CoreML (4-bit quantized) 178.71 11.05 0.1613

Interpretation

  • CoreML reduces mean latency and substantially reduces latency variance in this benchmark setup.
  • BLEU-4 decreases slightly under 4-bit quantization.

Notes / troubleshooting

  • CoreML conversion is most reliable with a Python 3.11/3.12 environment.
  • If ONNX→CoreML conversion fails in your local stack, this repo supports a conversion mode that handles conversion from wrappers.
