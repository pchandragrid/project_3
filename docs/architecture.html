System Architecture

A deep dive into the Transformer models, training pipeline, and deployment strategy powering the Image Captioning system.

The Models

We leverage three distinct state-of-the-art architectures to compare performance and trade-offs.

1. BLIP (Bootstrapping Language-Image Pre-training)

Architecture: Vision Transformer (ViT) encoder + BERT-based text decoder.

BLIP is designed for unified vision-language understanding and generation. It effectively bootstraps from noisy web data by filtering and re-captioning, leading to superior performance on COCO.

  • Strengths: High CIDEr scores, robust to diverse inputs.
  • Use Case: Primary model for high-quality captions.

2. ViT-GPT2

Architecture: Vision Transformer (ViT) encoder + GPT-2 decoder.

This is a classic encoder-decoder architecture where the image is encoded by a ViT and then fed into a GPT-2 language model via cross-attention layers.

  • Strengths: Fast inference, leverages strong language modeling of GPT-2.
  • Weaknesses: Can hallucinate details not present in the image.

3. GIT (Generative Image-to-text Transformer)

Architecture: A single unified Transformer.

GIT treats the image as a sequence of patch tokens and concatenates them with text tokens, processing everything in a single transformer block. This simplifies the architecture significantly.

  • Strengths: Simple, unified architecture.
  • Use Case: Good baseline for comparison with specialized V-L models.

Training Pipeline

The project implements a rigorous training pipeline to fine-tune these models on the MS COCO dataset.

Phase 1: Basic Fine-tuning

Initial training on a subset of COCO captions (10k-50k samples).

  • Goal: Adapt the pre-trained weights to the specific style of COCO captions.
  • Resolution: 224x224 pixels (memory efficiency).
  • Loss: Standard Cross-Entropy Loss (Language Modeling).

Phase 2: Advanced Optimization (BLIP)

A more sophisticated training loop implemented in src/training/train_phase2.py.

  • Data Filtering: Uses COCODatasetAdvanced to remove low-quality, repetitive, or extremely short captions.
  • Metric-Aware Training: Evaluates CIDEr scores on the validation set at the end of each epoch.
  • Early Stopping: halts training if validation CIDEr stops improving to prevent overfitting.
  • Gradient Checkpointing: Enabled to fit larger batch sizes on GPU/MPS memory.

Deployment Architecture

The system is designed for easy deployment and scalability using Hugging Face Spaces.

Model Hosting (Hugging Face Hub)

Trained model weights are not stored in the Git repository. Instead, they are pushed to the Hugging Face Model Hub:

  • pchandragrid/blip-caption-model
  • pchandragrid/vit-gpt2-caption-model
  • pchandragrid/git-caption-model

This keeps the codebase lightweight and allows the app to download models on-demand.

Frontend (Streamlit)

The user interface is built with Streamlit (apps/app.py). It handles:

  • Image upload and preprocessing.
  • Model selection and inference execution.
  • Result visualization (captions, confidence scores).
